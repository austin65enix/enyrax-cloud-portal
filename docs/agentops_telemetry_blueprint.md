# AgentOps Telemetry Blueprint

## 1. Product Positioning

AgentOps is the ENYRAX operations module for observing AI Agent workload, cost, efficiency, outcomes, and risk.

SOC tracks security incidents.
ServiceOps tracks human work orders and ownership.
ProjectOps tracks project progress.
Vulnerability Inventory tracks vulnerability risk.
AgentOps tracks AI Agent task execution, Token cost, tool usage, and delivered outcomes.

## 2. First Data Source

The first phase data source is local Codex session telemetry:

```text
~/.codex/sessions/*.jsonl
```

These files can be treated as a local AI agent telemetry source.

Phase one should not read or store full prompt / response content. It should only extract statistical metadata to avoid exposing sensitive content.

## 3. Core Question

我的 Codex 今天到底幫我做了多少工作？

The first AgentOps dashboard should break that question into operational questions:

* 今天跑了幾個 agent session？
* 總共消耗多少 tokens？
* cached tokens 佔比多少？
* output tokens 多少？
* reasoning tokens 多少？
* 哪些 task 最重？
* 哪些 project 最常被操作？
* 修改了哪些檔案？
* 成功率多少？
* 哪些 session 失敗或中斷？
* 平均任務時間多久？

## 4. Suggested Data Model

Planned table:

```sql
agent_runs
```

Columns:

* id
* session_id
* source
* project_name
* task_name
* task_number
* started_at
* ended_at
* duration_seconds
* status
* result
* model
* input_tokens
* cached_tokens
* output_tokens
* reasoning_tokens
* total_tokens
* tool_calls
* files_modified
* commands_run
* error_count
* warning_count
* created_at

Notes:

* `source` can be `codex_local` in the first version.
* `status` can be `success`, `failed`, `interrupted`, or `unknown`.
* `result` should store a short summary, not full conversation content.

## 5. Privacy and Safety Boundary

Phase one AgentOps does not save full prompt, response, secret, API key, password, or `.env` content.

Only save:

* session metadata
* token usage
* duration
* tool counts
* modified file count
* task name
* project name
* result summary

Avoid saving:

* raw conversation
* private notes
* credentials
* environment variables
* full shell output
* file contents

## 6. Local Parser Concept

Future parser:

```text
scripts/parse_codex_sessions.py
```

Functions:

* Scan `~/.codex/sessions/*.jsonl`
* Parse session metadata
* Extract token usage
* Extract tool call count
* Extract files modified count
* Infer task number, for example `Task #096`
* Infer project name, for example `Vulnerability Inventory`, `ServiceOps`, or `Sync Gateway`
* Output normalized JSON:

```text
data/agentops/agent_runs.json
```

## 8. Metrics Definitions

* Agent Runs: completed or attempted agent sessions
* Total Tokens: input + output + reasoning and other counted Token usage
* Cached Tokens: context cache reused tokens
* Output Tokens: tokens generated by the AI
* Reasoning Tokens: reasoning / thinking tokens
* Tool Calls: tool calls such as file reads, file edits, command execution, and search
* Files Modified: number of files modified by the session
* Success Rate: successful sessions / all sessions
* Avg Duration: average session duration
