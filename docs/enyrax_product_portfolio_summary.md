# ENYRAX Product Portfolio Summary

Interview and demo opening summary for the ENYRAX operations platform

## One-line Positioning

ENYRAX is an operations platform that connects SOC, ServiceOps, ProjectOps, AgentOps, Team_AgentOps, and Plan_ServiceOPS into one governed command center for human teams and AI agents.

ENYRAX 是一套把 SOC、ServiceOps、ProjectOps、AgentOps、Team_AgentOps 與 Plan_ServiceOPS 串成同一個治理型戰情中心的營運平台，讓人類團隊與 AI Agent 能在同一套流程中協作、追蹤、審核與交付。

ENYRAX turns scattered operations, security events, service tickets, projects, and AI agent work into one traceable delivery platform.

ENYRAX 把分散的資安事件、維運工單、專案進度與 AI Agent 工作紀錄，整合成可追蹤、可審核、可交付的平台。

## Executive Summary

ENYRAX is not a single dashboard or a single ticketing system. It is an integrated operations platform that connects security, service operations, project delivery, daily planning, AI agent governance, and audit evidence.

* SOC handles security incidents and risk.
* ServiceOps manages operational tickets and service requests.
* ProjectOps manages projects, timelines, and delivery progress.
* Plan_ServiceOPS brings personal daily tasks, team queue items, and project deadlines into a daily work command center.
* AgentOps manages AI Agent execution records, snapshots, risk, review, and release readiness.
* Team_AgentOps manages human-AI team collaboration, project contribution, human review, and accountability boundaries.
* Audit / Status provides operation trails, health checks, and the platform trust foundation.

ENYRAX 的價值不是單一功能，而是把企業裡原本分散的事件、工單、專案、AI Agent 執行紀錄與審核流程接回同一個營運脈絡。它讓主管可以看到進度，讓工程師可以留下證據，讓 AI Agent 的產出可以被審核，也讓整個交付流程更可追蹤。

## Module Summaries

### SOC

**Purpose**

SOC tracks security incidents, risk level, alert context, and response progress.

SOC 用來追蹤資安事件、風險等級、告警脈絡與處理進度。

**What it shows**

* Open incidents
* Critical / high severity events
* Incident workflow
* Evidence and closure readiness

**Demo talking point**

SOC is where raw security alerts become operational incidents that can be assigned, tracked, verified, and closed.

SOC 的價值是把原始資安告警轉成可指派、可追蹤、可驗證、可結案的營運事件。

### ServiceOps

**Purpose**

ServiceOps manages operational tickets, service requests, worklogs, progress, and closure flow.

ServiceOps 管理維運工單、服務請求、處理進度、工作紀錄與結案流程。

**What it shows**

* Ticket queue
* Pending / Doing / Done state
* Worklog and progress
* Archive / restore workflow
* Role-aware operations

**Demo talking point**

ServiceOps turns daily IT operations into a structured workflow instead of scattered messages or undocumented work.

ServiceOps 把日常 IT 維運從分散訊息與口頭交辦，整理成有狀態、有紀錄、有結案的工作流程。

### ProjectOps

**Purpose**

ProjectOps tracks project portfolio, milestones, owners, timelines, and delivery status.

ProjectOps 追蹤專案組合、里程碑、負責人、時程與交付狀態。

**What it shows**

* Project list
* Start date / end date
* Project status
* Portfolio overview
* Risk and deadline visibility

**Demo talking point**

ProjectOps connects operational work back to project delivery, so tickets are not isolated tasks but part of a larger delivery plan.

ProjectOps 讓工單不只是零散任務，而是能回到專案時程與交付計畫裡。

### Plan_ServiceOPS

**Purpose**

Plan_ServiceOPS is a personal daily work queue dashboard for IT operators.

Plan_ServiceOPS 是給維運人員每天打開使用的個人工作中控台。

**What it shows**

* Today tickets
* Team queue
* Project deadlines
* API-backed dashboard data
* Demo fallback when API is unavailable

**Demo talking point**

Plan_ServiceOPS answers the operator’s morning question: what should I handle first today?

Plan_ServiceOPS 回答維運人員每天早上的問題：今天我應該先處理什麼？

### AgentOps

**Purpose**

AgentOps provides observability and governance for AI agent work.

AgentOps 提供 AI Agent 工作紀錄的可觀察性與治理能力。

**What it shows**

* Preview records
* Token estimates
* Review status
* Release quality score
* Risk and anomaly indicators
* Historical snapshots
* Retention health

**Demo talking point**

AgentOps turns AI agent activity from invisible chat history into observable, reviewable, and governable operational data.

AgentOps 把原本看不見的 AI 對話與執行痕跡，轉換成可觀察、可審核、可治理的營運資料。

**Safety note**

AgentOps uses safe metadata boundaries and does not treat token estimates as billing-grade cost data.

AgentOps 保留 safe metadata 邊界，token 數值是營運估算，不是 billing-grade 成本資料。

### Team_AgentOps

**Purpose**

Team_AgentOps manages how human teams and AI agents collaborate on projects, tickets, reviews, and delivery outcomes.

Team_AgentOps 管理人類團隊與 AI Agent 如何在專案、工單、審核與交付結果上協作。

**What it shows**

* Active agents
* Pending reviews
* Project impact
* Agent activity timeline
* Project agent contribution
* Human review flow
* Team scorecard
* Shadow AI risk

**Demo talking point**

Team_AgentOps makes AI agents manageable, auditable, and measurable contributors within enterprise teams.

Team_AgentOps 讓 AI Agent 成為企業團隊中可管理、可稽核、可衡量貢獻的正式成員。

**Safety note**

Team_AgentOps is for governance and protection, not employee surveillance scoring.

Team_AgentOps 的目標是治理與保護，不是員工監控或績效懲罰。

### Audit / Status

**Purpose**

Audit and Status provide operation trails, platform health, and trust signals.

Audit 與 Status 提供操作軌跡、平台健康狀態與信任訊號。

**What it shows**

* Create / update / delete operation trail
* API health
* Module status
* Deployment confidence
* Evidence for review

**Demo talking point**

Audit and Status make the platform explainable and trustworthy, especially when multiple modules and AI-assisted workflows are involved.

當多個模組與 AI 協作流程串在一起時，Audit 與 Status 讓平台更可解釋、更可信任。

## How the Modules Work Together

```text
SOC detects or receives a security event
→ ServiceOps creates or tracks the response ticket
→ ProjectOps links the work to project or infrastructure impact
→ Plan_ServiceOPS shows the operator what to handle today
→ AgentOps observes AI agent work used during analysis, coding, documentation, or review
→ Team_AgentOps connects AI agent contribution back to project progress and human review
→ Audit / Status provides operation evidence and system health
```

```text
SOC 接收資安事件
→ ServiceOps 建立或追蹤處理工單
→ ProjectOps 將工作連回專案或基礎設施影響
→ Plan_ServiceOPS 告訴維運人員今天要先處理什麼
→ AgentOps 觀測 AI Agent 在分析、開發、文件或審核中的工作紀錄
→ Team_AgentOps 將 AI Agent 的貢獻連回專案進度與人工審核
→ Audit / Status 提供操作證據與系統健康狀態
```

ENYRAX is an enterprise operations command center that connects security incidents, operational tickets, project progress, and AI agent work into one traceable workflow.

In many companies, SOC, ticketing, and project management are separated, while AI agents are often used privately by individuals. ## Product Value Summary

### For Managers

* 看見 SOC、工單、專案與 AI Agent 對交付的實際影響。
* 掌握進度、風險、卡點、review pending。
* 不只知道「有使用 AI」，而是知道 AI 對專案貢獻了什麼。

### For Engineers / Operators

* 每日知道優先處理項目。
* 工單、專案、AI 產出都有紀錄。
* AI-assisted work 有 review、有證據、有責任邊界。
* 避免出問題時被簡化成「是不是你亂用 AI」。

### For Governance / Audit

* 保留操作紀錄。
* 保留 AI Agent 使用與 review 邊界。
* 降低 shadow AI。
* 提供平台健康與稽核證據。

### For Delivery

* 將 incidents、tickets、projects、agent outputs 串成交付流程。
* 讓 AI Agent 不只是工具，而是可管理的交付資源。
* 讓工作從分散 dashboard 走向可治理 workflow。

## Safety and Boundary Statement

* AgentOps / Team_AgentOps follow a safe metadata approach.
* No full prompt / response storage.
* No raw session storage.
* No credentials / secrets storage.
* Token values are operational estimates, not billing-grade cost data.
* Team scorecard is a governance metric, not an employee surveillance score.
* AI Agent output still requires human review.
* ENYRAX emphasizes traceability, reviewability, and accountability.

ENYRAX 的 AI Agent 管理不是為了監控員工，而是為了讓 AI 協作可追蹤、可審核、可交付，也保護使用者與團隊。
